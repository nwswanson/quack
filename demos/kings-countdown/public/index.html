The Lost Update Countdown

This room is trying to count to 1000. Every browser joins, presses Say go, then starts sending random chunks between 1 and 20. If the remaining distance is less than 20, the browser sends exactly the remaining amount.

In Locked mode, the backend wraps the read-modify-write in ctx.locks().acquire(...). Each chunk sees the latest total, so the counter walks cleanly to 1000.

In Unsafe mode, the backend intentionally reads the total, does a little busy work, and writes the new value without a lock. Concurrent websocket handlers can read the same old total and overwrite each other. The room still receives many chunks, but some of the work disappears.

The applied number counts all chunk work the backend accepted. The visible total is the shared room value. In unsafe mode, lost grows when stale writes erase accepted work. 

before = memory.get("total")
# another handler can write here
memory.set("total", before + amount)

Open a few browser windows, choose Unsafe, and press go. Then reset into Locked mode and watch the same noisy clients behave.
