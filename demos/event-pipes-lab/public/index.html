Event Pipes Lab

Three websocket workflows built from hardcoded pipe edges, named Starlark handler nodes, and session-scoped websocket traces.

Connecting

Map Reduce

The client publishes one fixed start edge. The splitter partitions text across four explicit map workers, then the reducer merges those worker outputs.

  1. pipe-demo.map_reduce.start -> api/map_reduce.star:start_node
  2. pipe-demo.map_reduce.split -> api/map_reduce.star:split_node
  3. pipe-demo.map_reduce.map_0..3 -> explicit map worker nodes
  4. pipe-demo.map_reduce.reduce -> api/map_reduce.star:reduce_node
Run the pipeline to see word counts.

Scatter/Gather

A controller scatters the request to four independent service nodes. Each service returns a partial answer, then the gather node merges them.

  1. pipe-demo.scatter_gather.start -> api/scatter_gather.star:start_node
  2. profile, pricing, inventory, risk service edges
  3. pipe-demo.scatter_gather.gather -> api/scatter_gather.star:gather_node
Run the pipeline to see gathered worker scores.

Sharding

The router hashes records onto four fixed shard edges. Each shard node computes a local count and sum, then the merge node combines the summaries.

  1. pipe-demo.sharding.route -> api/sharding.star:route_node
  2. pipe-demo.sharding.shard_0..3 -> explicit shard nodes
  3. pipe-demo.sharding.merge -> api/sharding.star:merge_node
Run the pipeline to see shard totals.
