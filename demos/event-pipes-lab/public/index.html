Event Pipes Lab

Three websocket workflows built from pipe events, stage handlers, and session-scoped channels.

Connecting

Map Reduce

The client publishes a start event. The server splits text into chunks, maps each chunk into word-count pairs, then reduces those pairs into final totals.

  1. pipe-demo.map_reduce.<session>.start
  2. split publishes several map events
  3. reduce publishes the websocket-visible result

Scatter/Gather

A controller scatters independent work items to worker events. Each worker returns a partial score, and the gather stage waits until the session has every response.

  1. start fans out to many worker pipe events
  2. Workers store partial responses under the session id
  3. gather merges responses and chooses the highest score

Sharding

The router hashes records onto shard events. Every shard computes its local count and sum, then a merge event combines the shard summaries.

  1. route assigns records by stable key hash
  2. shard handlers process separate partitions
  3. merge shows the aggregate result
